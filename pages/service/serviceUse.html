Ⅰ OPENAPI 서비스 신청 안내

1. 데이터공개 ( 분류별, 제공기관별, 유형별 ) 버튼 클릭

2. 원하는 Open-API <서비스>의 선택박스 체크 후 우측 상단의 Open-API 이용신청 버튼 클릭

(※ Open-API 서비스 이용을 위해서는 반드시 회원가입 및 로그인이 완료된 상태여야 합니다.)

  • 1 원하는 서비스의 선택박스 체크(중복 선택 가능)
  • 2 체크항목 Open-API 이용신청 버튼 클릭

3. Open-API 이용신청 시 공공데이터활용 이용약관 동의

  • 1 이용약관 동의 체크

4. 선택한 OPENAPI 서비스의 세부항목 입력 후 신청 버튼 클릭

  • 1 활용용도 선택 (어플리케이션 개발/웹사이트 연동 및 운영/기타 중 택 1)
  • 2 활용구분 선택 (개인/기업 중 택 1, 기업 선택 시 기업명 입력)
  • 3
    활용목적 작성 (상세 입력 요망)
    예시1) 영양성분DB 정보를 활용한 다이어트 어플리케이션 개발
    예시2) 협회 회원사 간 학술활동 자료로 활용
  • 4 신청 버튼 클릭
  • 5 목록 화면 호출 시 목록 버튼 클릭

5. 인증키 발급 정보 및 서비스신청 목록 확인

  • 1 자신이 신청한 인증키 발급정보 확인
  • 2 자신이 신청한 Open-API 신청 목록 확인
※ 인증키는 신청 후 자동으로 승인되어 발급이 완료됩니다. (인증키 발급 및 서비스 신청 종료)

Ⅱ OPENAPI 서비스 확인 및 활용 안내

1. 인증키 신청 현황 버튼 클릭

2. 인증키 발급 정보 및 서비스 신청 목록 확인

  • 1 자신이 신청한 인증키 발급정보 확인(인증키 발급은 1회만 가능 함)
  • 2 자신이 신청한 Open-API 신청 목록 확인(동일한 서비스는 1회만 신청하면 이용 가능)

3. 원하는 정보 찾기

  • 1 메인화면 상단의 데이터 공개 클릭(분류별, 제공기관별, 유형별 중 택 1)
  • 2 서비스 명에 자신이 찾고자 하는 공공데이터 정보를 검색
  • 3 다양한 검색 조건을 통해 원하는 공공데이터 서비스 조회 가능(서비스명 클릭 시 상세화면으로 이동)
  • 4 메타 정보 확인
  • 5 SAMPLE 정보 확인
  • 6 Open-API 정보 확인

4. 발급된 인증키를 이용하여 Open-API 사용

  • 1 요청주소(URL, 샘플) 확인

5. 요청주소 입력 상세설명

  • 1 keyID : 자신의 인증키 발급정보 입력
  • 2 serviceID : 미리보기 주소에 있는 serviceID 입력
  • 3 datatype : XML, JSON 중 택1 하여 입력
  • 4 startIdx : 받고자 하는 정보의 시작 행 순번 입력 (1번부터 받으실 거라면 ‘1’ 입력)
  • 5 endIdx : 받고자 하는 정보의 마지막 행 순번 입력(5번까지 받으실 거라면 ‘5’ 입력)
요청 주소 입력 예시 http://openapi.foodsafetykorea.go.kr/api/본인인증키/I0960/xml/1/5

이해하기

이 문서는 식품안전나라에서 제공하는 공공데이터(Open API)를 활용하기 위한 기본 개념과 사용 방법을 안내합니다. 오픈 API를 통해 식약처의 식품안전정보를 여러분의 서비스에 손쉽게 통합할 수 있습니다.

특징

  • 다양한 데이터 포맷 지원 XML, JSON 두 가지 형태의 응답 포맷을 지원하여 개발 환경에 맞춰 유연하게 사용할 수 있습니다.
  • RESTful 아키텍처 기반 직관적이고 표준화된 HTTP URL 방식을 사용하여 접근성이 뛰어납니다.

기본 정보

REST API를 호출하기 위한 기본 도메인과 방식은 다음과 같습니다.

GET http://openapi.foodsafetykorea.go.kr/api/{keyId}/{serviceId}/{dataType}/{startIdx}/{endIdx}

요청 파라미터

이름 타입 필수 여부 설명
keyId String O 발급받은 인증키
serviceId String O 서비스명 (예: I1250)
dataType String O 응답형식 (XML / JSON)

요청 예시

JSON 포맷으로 1번부터 5번까지의 데이터를 요청하는 예시입니다.

curl -v -X GET "http://openapi.foodsafetykorea.go.kr/api/sample/I2500/json/1/5"
