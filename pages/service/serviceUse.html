(※ Open-API 서비스 이용을 위해서는 반드시 회원가입 및 로그인이 완료된 상태여야 합니다.)

  • 1원하는 서비스의 선택박스 체크(중복 선택 가능)
  • 2체크항목 Open-API 이용신청 버튼 클릭

3. Open-API 이용신청 시 공공데이터활용 이용약관 동의

4. 선택한 OPENAPI 서비스의 세부항목 입력 후 신청 버튼 클릭

  • 1활용용도 선택 (어플리케이션 개발/웹사이트 연동 및 운영/기타 중 택 1)
  • 2활용구분 선택 (개인/기업 중 택 1)
  • 3활용목적 작성 후 신청 버튼 클릭

5. 인증키 발급 정보 및 서비스신청 목록 확인

※ 인증키는 신청 후 자동으로 승인되어 발급이 완료됩니다.

회원가입 및 권한 신청

파일 다운로드는 회원가입 없이도 가능합니다. Open-API 인증키가 필요한 경우에만 아래 절차를 따르세요.

1단계 — 식품안전나라 접속 후 회원가입

www.foodsafetykorea.go.kr에 접속하여 우측 상단 [회원가입] 버튼을 클릭합니다. 이메일 인증 후 가입이 완료됩니다.

이미 식품안전나라 회원이라면 이 단계를 건너뛰세요.

2단계 — 공공데이터 활용 포털 이동

상단 메뉴 데이터공개 → Open-API를 클릭하거나, 공공데이터활용 포털(apiMain.do)로 직접 이동합니다.

3단계 — 원하는 서비스 선택 후 이용 신청

필요한 데이터 서비스에 체크 표시한 뒤 [Open-API 이용신청] 버튼을 클릭합니다. 활용 목적을 입력하면 인증키가 즉시 발급됩니다.

인증키는 1회 발급이며, 마이페이지 → 인증키 신청 현황에서 언제든지 확인할 수 있습니다.

원하는 데이터 찾기

식품안전나라는 178종의 공공데이터를 제공합니다. 아래 방법으로 필요한 데이터를 빠르게 찾을 수 있습니다.

A

키워드 검색

포털 검색창에 "건강기능식품", "위생등급", "영양성분" 등 키워드를 입력하면 관련 데이터가 목록으로 표시됩니다.

B

분류별 탐색

식품 제조·가공 / 수입식품 / 건강기능식품 / 축산물 / 기준·규격 등 주제별 카테고리로 탐색할 수 있습니다.

C

데이터맵 활용

이 플랫폼의 데이터맵에서 테이블 간 연결 관계를 시각적으로 확인하면서 원하는 데이터를 찾을 수 있습니다.

자주 묻는 질문

인증키 없이도 데이터를 받을 수 있나요?

네. 파일 다운로드(엑셀·CSV)는 회원가입 없이도 가능합니다. Open-API를 통해 실시간으로 데이터를 호출하려면 인증키가 필요합니다.

인증키는 무료인가요?

네, 무료입니다. 회원가입 후 이용 신청만 하면 즉시 발급되며, 별도 비용이 없습니다.

하루에 몇 번까지 API를 호출할 수 있나요?

기본 제공 호출 횟수는 일 1,000건이며, 사업자의 경우 확장 신청이 가능합니다. 자세한 사항은 공공데이터포털 이용약관을 참고하세요.

데이터를 상업적으로 활용해도 되나요?

공공누리 제4유형에 해당하지 않는 데이터는 출처 표시 조건으로 상업적 활용이 가능합니다. 개별 데이터셋의 이용 조건을 반드시 확인하세요.

이해하기

식품안전나라 Open API는 식약처의 식품안전정보를 RESTful 방식으로 제공하는 공공 데이터 서비스입니다. XML·JSON 두 가지 포맷을 지원하며, 178종의 서비스를 무료로 활용할 수 있습니다.

RESTful + HTTP GET

URL 파라미터 방식이라 별도 SDK 없이 모든 언어에서 바로 호출 가능합니다.

XML / JSON 지원

요청 URL의 dataType 파라미터로 원하는 포맷을 선택할 수 있습니다.

페이지네이션 내장

startIdx / endIdx로 레코드 범위를 지정하는 오프셋 기반 페이징을 사용합니다.

무료 · 즉시 발급

회원가입 후 이용 신청 즉시 인증키가 발급됩니다. 별도 심사 없음.

Base URL

http://openapi.foodsafetykorea.go.kr/api/

인증키 발급 절차

  1. 1
    식품안전나라 로그인

  2. 2

  3. 3

    인증키 확인

API 레퍼런스 › REST API 가이드

REST API 가이드

요청 URL

GET http://openapi.foodsafetykorea.go.kr/api/{keyId}/{serviceId}/{dataType}/{startIdx}/{endIdx}

요청 파라미터

파라미터타입필수설명
keyIdStringY발급받은 인증키
serviceIdStringY서비스 고유 ID (예: I1250, I0030)
dataTypeEnumYjson 또는 xml
startIdxIntegerY조회 시작 행 번호 (1부터 시작)
endIdxIntegerY조회 끝 행 번호 (최대 1,000)

요청 예시

# JSON — 식품제조가공업정보 1~10건
curl -X GET "http://openapi.foodsafetykorea.go.kr/api/YOUR_KEY/I1220/json/1/10"

# XML — 건강기능식품 품목 1~5건
curl -X GET "http://openapi.foodsafetykorea.go.kr/api/YOUR_KEY/I0030/xml/1/5"

# JavaScript (fetch)
const res = await fetch(
  `http://openapi.foodsafetykorea.go.kr/api/YOUR_KEY/I0030/json/1/10`
);
const data = await res.json();
console.log(data.I0030.row);

Python 예시

import requests

KEY = "YOUR_API_KEY"
SERVICE = "I0030"  # 건강기능식품 품목

url = f"http://openapi.foodsafetykorea.go.kr/api/{KEY}/{SERVICE}/json/1/100"
response = requests.get(url)
data = response.json()

rows = data[SERVICE]["row"]
for item in rows:
    print(item["PRDLST_NM"], item["PRIMARY_FNCLTY"])

데이터 포맷 안내

JSON 응답 구조

{
  "I0030": {
    "total_count": "12345",   // 전체 데이터 건수
    "row": [
      {
        "LCNS_NO": "20220001",
        "BSSH_NM": "홍길동식품(주)",
        "PRDLST_NM": "비타민C 1000",
        "PRIMARY_FNCLTY": "비타민 C는 결합조직 형성과 기능 유지에...",
        "PRDLST_CDNM": "비타민류",
        "POG_DAYCNT": "180"
      },
      ...
    ],
    "RESULT": {
      "MSG": "정상 처리되었습니다.",
      "CODE": "INFO-000"
    }
  }
}
페이지네이션 계산법
다음 페이지: startIdx = endIdx + 1. 전체 건수는 total_count를 활용하세요.

XML 응답 구조

<?xml version="1.0" encoding="UTF-8"?>
<I0030>
  <total_count>12345</total_count>
  <row>
    <LCNS_NO>20220001</LCNS_NO>
    <BSSH_NM>홍길동식품(주)</BSSH_NM>
    <PRDLST_NM>비타민C 1000</PRDLST_NM>
  </row>
  <RESULT>
    <MSG>정상 처리되었습니다.</MSG>
    <CODE>INFO-000</CODE>
  </RESULT>
</I0030>

인코딩 / 주의사항

  • 응답 인코딩: UTF-8
  • 한 번에 최대 1,000건까지 조회 가능
  • 빈 값은 null 또는 빈 문자열로 반환
  • 날짜 컬럼 포맷: YYYYMMDD
API 레퍼런스 › 오류 코드

오류 코드

응답 본문의 RESULT.CODE 필드로 처리 결과를 확인할 수 있습니다.

코드유형메시지해결 방법
INFO-000 성공 정상 처리되었습니다. 정상 응답
ERROR-001 인증 오류 인증키가 유효하지 않습니다. keyId 값 확인 및 재발급 신청
ERROR-002 권한 오류 해당 서비스 이용 권한이 없습니다. 해당 serviceId에 대한 이용 신청 필요
ERROR-003 파라미터 요청 파라미터가 올바르지 않습니다. startIdx ≤ endIdx, endIdx - startIdx ≤ 999 확인
ERROR-004 서비스 해당 서비스가 존재하지 않습니다. serviceId 오탈자 확인
ERROR-010 트래픽 일일 호출 한도를 초과하였습니다. 다음 날 재시도 또는 호출 한도 확장 신청
ERROR-500 서버 서버 내부 오류입니다. 잠시 후 재시도, 지속 시 문의
오류 처리 패턴: RESULT.CODE가 INFO-000이 아니면 row 배열이 없을 수 있습니다. 반드시 분기 처리하세요.

실시간 위해식품 알림 대시보드

I0490(회수·판매중지) + I2620(검사부적합)를 JOIN하여 최신 위해식품 현황을 보여주는 대시보드입니다.

// 회수·판매중지 최신 10건 fetch
const [recall, fail] = await Promise.all([
  fetch(`/api/KEY/I0490/json/1/10`).then(r => r.json()),
  fetch(`/api/KEY/I2620/json/1/10`).then(r => r.json())
]);
const rows = [
  ...recall.I0490.row.map(r => ({...r, type:'회수'})),
  ...fail.I2620.row.map(r => ({...r, type:'부적합'}))
].sort((a, b) => b.RTRVL_DCLS_DE - a.RTRVL_DCLS_DE);

식단 관리 앱 영양 DB 연동

1471000(식품영양성분 DB) + COOKRCP01(조리식품 레시피)를 활용해 식단 입력 → 자동 영양 계산을 구현합니다.

# 음식명으로 영양성분 검색 (React Native)
const search = async (foodName) => {
  const url = `http://openapi.foodsafetykorea.go.kr/api/`
    + `${KEY}/1471000/json/1/20`;
  const res = await fetch(url);
  const { "1471000": data } = await res.json();
  return data.row.filter(r =>
    r.DESC_KOR.includes(foodName)
  );
};
